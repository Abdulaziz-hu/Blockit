Privacy Policy

BlockIt Chrome Extension

Last Updated: February 18, 2026

BlockIt (“we”, “our”, or “the extension”) is committed to protecting your privacy. This Privacy Policy explains how BlockIt handles your data.

1. Data Collection

BlockIt does NOT collect, transmit, or share any personal data.

All data is stored locally on your device using Chrome's storage API. This includes:

  • Blocked Sites List: Domain names you've chosen to block
  • Extension Settings: Your preferences (theme, global toggle state)
  • Usage Statistics: Local counters (number of times blocked, install date)
  • Break Times: Temporary timestamps for break time features

None of this data leaves your device.

2. Data Storage

All data is stored using:

  • chrome.storage.local — Persistent local storage on your device
  • chrome.storage.session — Temporary session storage (cleared when browser closes)

This data is:

  • ✓ Stored only on your local device
  • ✓ Never transmitted to any server
  • ✓ Never shared with third parties
  • ✓ Not synced across devices
  • ✓ Completely under your control

3. Network Requests

BlockIt makes one optional network request:

  • Update Checker: Checks GitHub's public API for new releases
    URL: https://api.github.com/repos/Abdulaziz-hu/blockit/releases/latest
    Frequency: On install and once per day
    Data Sent: None — This is a public API call with no identifiable information

No tracking, analytics, or telemetry of any kind.

4. Permissions Explained

BlockIt requires the following Chrome permissions:

declarativeNetRequest
Why: To redirect blocked sites to the blocked page
Usage: Creates blocking rules that run locally in your browser
storage
Why: To save your blocked sites list and preferences
Usage: Stores data locally on your device only
tabs
Why: To block the current tab and redirect after unblocking
Usage: Reads current tab URL when you click “Block this site”
activeTab
Why: To interact with the currently active tab
Usage: Allows quick-blocking of the current page
webNavigation
Why: To detect when you navigate to a blocked site
Usage: Monitors navigation to apply blocking rules
host: <all_urls>
Why: To block any website you choose
Usage: Allows blocking rules to work on all domains

Important: These permissions are used exclusively for blocking functionality. BlockIt does not read page content, access passwords, or collect browsing history.

5. Third-Party Services

BlockIt does not use any third-party services, analytics, or tracking tools.

The only external interaction is the optional update check to GitHub's public API (see section 3).

6. Data Deletion

You have complete control over your data:

  • Delete individual sites: Click the × button next to any site
  • Clear all data: Settings → “Clear All Data” button
  • Uninstall extension: Removes all stored data automatically

Since all data is stored locally, deleting the extension or clearing its data permanently removes everything.

7. Children's Privacy

BlockIt does not collect any data from anyone, including children under 13. The extension can be safely used by people of all ages.

8. Changes to This Policy

If we make changes to this Privacy Policy, we will update the “Last Updated” date at the top. Continued use of BlockIt after changes constitutes acceptance of the updated policy.
